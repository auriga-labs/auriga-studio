### 2-2. モニター描画のモード遷移

中央プレビュー（モニター）は2モードを持ち、操作に応じて自動遷移する。

```
        メディアをクリック / 右クリック「モニターに表示」
   ┌──────────────────────────────────────────────┐
   ▼                                              │
┌─────────────────┐   クリップ選択 / 再生 / シーク   ┌─────────────────┐
│  source モード   │ ─────────────────────────────▶ │ program モード   │
│ 単一メディアを    │                                │ タイムライン全     │
│ <video>/背景画像 │ ◀───────────────────────────── │ レイヤーを canvas  │
│ で表示           │   （メディアクリックで戻る）      │ に合成 (composite) │
└─────────────────┘                                └─────────────────┘
```

- `program`: `composite(time)` が下位レイヤー→上位レイヤーの順に `canvas` へ描画。映像・画像・テキストを合成し、動画/音声は `syncAV()` で再生位置に同期。
- `source`: メディアプールの 1 ソースをそのまま表示（プレビュー確認用）。

### 2-3. テーマ切り替え機構

配色（`themes/<name>.css`）と挙動（`themes/<name>.js`）を分離。

```
applyTheme(name)
  ├─ #themeLink の href を themes/<name>.css に差し替え（配色切替）
  ├─ documentElement.dataset.theme = name（構造判定用）
  └─ applyThemeScript(name)
       ├─ 直前テーマの cleanup(ctx) を実行（DOM 状態を戻す）
       ├─ themes/<name>.js を動的 <script> 読み込み（多重防止）
       └─ registerTheme で登録された apply(ctx) を実行
            └─ ctx 経由で workspace タブ名・タイトル接尾辞を変更
```

テーマ別のワークスペースタブ:
| テーマ | タブ構成 |
|--------|---------|
| Aurigaオリジナル | 編集 / カラー / オーディオ / 書き出し |
| YMM4 | （CSS で非表示・DaVinciライク） |
| DaVinci | カット / 編集 / カラー / デリバー |
| Premiere | （premiere.js による） |

## 3. 実装済み機能 ✅

### 3-1. タイムライン編集

- **連番レイヤー方式**: YMM4 ライクに種類を設けず「レイヤー1〜5」。どのレイヤーにも映像・音声・テキストを配置可（上が手前に描画）。
- クリップの**ドラッグ移動 / 左右ハンドルでのリサイズ（トリミング）**。
- **分割（カットツール / 分割ボタン / `S`）** — 再生ヘッド位置でクリップを2分割。
- **削除**（ボタン / `Delete` / `Backspace`）。
- **複製（`Ctrl+D`）/ コピー（`Ctrl+C`）/ 貼り付け（`Ctrl+V`）**。
- ツール切替: 選択(`V`) / カット(`C`) / ハンド(`H`)。
- **ズーム**スライダー（ルーラー・クリップ幅が連動）。
- **ルーラー / 再生ヘッド（赤線）** — どこでも掴んでドラッグスクラブ可能。空白クリックでもシーク。
- 横スクロールで再生ヘッド連動、縦スクロールでレイヤーヘッダー連動。
- タイムライン領域の**高さをドラッグでリサイズ**（`#tlResizer`・localStorage 保存）。

### 3-2. メディア

- **読み込み**: ボタン / ファイル入力 / **OS からのドラッグ&ドロップ**（直接トラックへ配置可）。
- 対応形式: 動画(mp4/mov/webm/mkv)、音声(mp3/wav/m4a)、画像(png/jpg/gif/webp)。
- **サムネイル自動生成**（動画は中間フレームを canvas 描画）。
- **再生尺の自動取得**（`probeDuration`）でバッジ表示・クリップ長に反映。
- メディアの右クリックメニュー: モニター表示 / タイムライン追加 / 名前変更 / プールから削除。

### 3-3. プレビュー・再生

- **canvas コンポジター**による複数レイヤー合成プレビュー。
- 再生 / 一時停止（`Space`）、先頭・末尾、前後フレーム、**ループ**、音量。
- 動画・音声のタイムライン位置同期再生（`syncAV` / 再生速度・ミュート対応）。
- **解像度切替**（1080p / 縦型 / 正方形 / 4K）— canvas とアスペクト比に反映。
- **タイムコード表示**（HH:MM:SS:FF / 30fps）。
- 右クリック「現在フレームを保存」で PNG ダウンロード。

### 3-4. プロパティパネル

選択クリップに対しリアルタイム反映（合成へ即時反映）:
- 変形（位置 X/Y・スケール・回転）
- 不透明度
- 再生速度
- カラー補正（明るさ・コントラスト・彩度 / CSS filter）

### 3-5. レイヤー操作（トラックヘッダー右クリック）

- 表示/非表示トグル、ミュート、レイヤー内クリップ一括削除。
- 全レイヤー表示 / 他レイヤー非表示 / レイヤー設定初期化。

### 3-6. メニューバー（`menu_layout` 駆動）

- **YMM4 v4.53.0.9 の実機メニューを JSON から動的生成**（ファイル/編集/レイアウト/ツール/ヘルプ）。
- ネイティブ風サブメニュー（ホバー展開・画面端補正・separator/radio/checkbox/dynamic 対応）。
- ロゴメニュー（全テーマ共通）: About / 環境設定 / ショートカット / 更新確認 / 終了 など。

### 3-7. UI 全般

- **スプラッシュ画面**（最低700ms表示後フェードアウト）。
- **About モーダル**（version.json を読み込み表示・情報コピー）。
- **トースト通知**。
- 各種コンテキストメニュー（クリップ/メディア/タイムライン空白/トラック/モニター）。
- **レスポンシブ**（720px 以下でヘッダー操作をハンバーガーパネルへ実体移設）。
- **localStorage 永続化**: テーマ / 解像度 / 再生ヘッド位置 / タイムライン高さ / ログインユーザー / Drive 容量キャッシュ。

### 3-8. 認証（Google OAuth）— 実装済み（フロント+PHPバックエンド）

- アバタークリックでアカウントポップオーバー。
- `window.open` で Google 認可画面 → PHP `callback.php` がトークン交換 → `postMessage` でユーザー情報を返却。
- **CSRF 対策**（クライアント生成 state の `app.` プレフィックス判別 + 照合）、オリジン厳格検証。
- ログイン状態の永続化・復元、ログアウト（サーバーセッション破棄）。
- **Google Drive 容量取得**（`drive/v3/about`）→ Auriga Cloud 使用量ゲージに反映・キャッシュ。

### 3-9. テーマ

- Aurigaオリジナル / YMM4 / DaVinci / Premiere の **4 テーマ切替**（配色 + 挙動 + ワークスペースタブ + タイトル）。

## 5. 既知の技術的メモ・留意点

- `scripts/write-version.js` は **`fs.writeFileSync` がコメントアウト**されており、現状 version.json を自動生成しない（手動管理）。`electron`/`chromium` 等のフィールドも version.json に未出力。
- About モーダルの表示項目（Electron/Chromium/Node/V8/OS）は version.json に該当キーがあれば表示する設計だが、現 version.json には `version/commit/date` のみ。
- OAuth の `config.php` は **gitignore 管理（手動デプロイ）**。クライアント ID は公開情報として `main.js` に保持。
- アクセストークンはメモリ保持のみ（短命）。再起動後は容量キャッシュ表示で代替。
- Undo/Redo・プロジェクト保存が未実装のため、**現状は編集結果を永続化できない**（メディアは `URL.createObjectURL` の一時参照）。

## 6. 推奨される次の実装ステップ

1. **YMM4 `.ymmp`（JSON）読み込み/書き出し** — 最優先。`menu_layout` のインポート/エクスポート系メニューと接続。

4. **Undo/Redo の履歴スタック**。
5. **エフェクトの実適用ロジック**。
6. Auriga Cloud のストレージ実体設計（※スケーリング・コスト試算を伴うため要事前検討）。
