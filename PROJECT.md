### 2-2. モニター描画のモード遷移

中央プレビュー（モニター）は2モードを持ち、操作に応じて自動遷移する。

```text
        メディアをクリック / 右クリック「モニターに表示」
   ┌──────────────────────────────────────────────┐
   ▼                                              │
┌─────────────────┐   クリップ選択 / 再生 / シーク   ┌─────────────────┐
│  source モード   │ ─────────────────────────────▶ │ program モード   │
│ 単一メディアを    │                                │ タイムライン全     │
│ <video>/背景画像 │ ◀───────────────────────────── │ レイヤーを canvas  │
│ で表示           │   （メディアクリックで戻る）      │ に合成 (composite) │
└─────────────────┘                                └─────────────────┘
```

- `program`: `composite(time)` が下位レイヤー→上位レイヤーの順に `canvas` へ描画。映像・画像・テキストを合成し、動画/音声は `syncAV()` で再生位置に同期。
- `source`: メディアプールの 1 ソースをそのまま表示（プレビュー確認用）。

### 2-3. テーマ切り替え機構

配色（`themes/<name>.css`）と挙動（`themes/<name>.js`）を分離。

```text
applyTheme(name)
  ├─ #themeLink の href を themes/<name>.css に差し替え（配色切替）
  ├─ documentElement.dataset.theme = name（構造判定用）
  └─ applyThemeScript(name)
       ├─ 直前テーマの cleanup(ctx) を実行（DOM 状態を戻す）
       ├─ themes/<name>.js を動的 <script> 読み込み（多重防止）
       └─ registerTheme で登録された apply(ctx) を実行
            └─ ctx 経由で workspace タブ名・タイトル接尾辞を変更
```

テーマ別のワークスペースタブ:

| テーマ | タブ構成 |
| -------- | --------- |
| Aurigaオリジナル | 編集 / カラー / オーディオ / 書き出し |
| YMM4 | （CSS で非表示・DaVinciライク） |
| DaVinci | カット / 編集 / カラー / デリバー |
| Premiere | （premiere.js による） |

### 2-4. レンダリング方式（描画・音声再生・レイヤー合成エンジン）

現状のプレビューエンジンは **GPU合成やタイムライン事前レンダリングを行わない、
Canvas 2D による毎フレーム即時描画（イミディエイトモード）** で構成されている。
書き出し（ファイルエンコード）用のオフラインレンダラーはまだ存在しない（§6-3）。

#### 描画エンジン

- 使用しているのは `<canvas id="compositor">` 1枚に対する **Canvas 2D API のみ**。WebGL・GPUレイヤー合成は不使用。
- `canvas.width/height` はプロジェクト解像度そのもの（例: 1920×1080）を直接設定し、内部解像度＝出力解像度。DOM上の表示倍率は CSS 変数 `--ar-w`/`--ar-h` によるアスペクト比固定の拡縮のみで、描画自体には影響しない。
- オフスクリーン `<canvas>` はクロマキー処理用の縮小バッファ（クリップごとに `_ckCanvas` を1枚キャッシュ、最大640px幅）としてのみ使われ、レイヤーごとの中間バッファ合成は行っていない＝**全レイヤーを同一の`compCtx`へ直接重ね描きする**シングルバッファ方式。

#### フレーム駆動（再生ループ）

- `requestAnimationFrame` による `tick()` ループ。固定fpsではなく `performance.now()` の差分(dt)を `state.playhead` に加算する **可変フレームレート方式**（ディスプレイのリフレッシュレートに追従。YMM4のような固定fps書き出しの概念はプレビューには無い）。
- 停止中もシーク・プロパティ変更のたびに `composite()` を単発呼び出しし、その場で1フレームだけ再描画する（イベント駆動の静止画更新）。
- タイムコード表示（HH:MM:SS:FF）は30fps換算の表示用フレーム番号であり、実際の再生・合成は秒単位（浮動小数）で連続的に行われる。

#### レイヤー合成の仕組み — `composite(time, playing)`

1. 毎フレーム canvas 全面を黒 `#000` で塗りつぶしてから合成開始（背景は常に不透明・アルファ抜けなし）。
2. `TRACKS`（レイヤー1〜5、配列の先頭＝レイヤー1）を **先頭から順に走査＝奥から手前へ描画**。YMM4と同じく「番号が大きい（下の行の）レイヤーほど手前」になる仕様。
3. 各レイヤーにつき `activeClipOnTrack()` で再生ヘッド位置に重なるクリップを**1本だけ**選択する。同一レイヤー上でクリップが重複している場合は開始位置が最も遅いものが勝つ（後勝ち）。クロスフェードのような多重合成は無く、1レイヤー＝1クリップ同時描画が原則。
4. YMM4側で非表示指定（`IsHidden`）のクリップは描画・音声同期とも完全スキップ。レイヤーヘッダーの目玉アイコン（`trackVisible`）は**描画のみ**を止め、`syncAV()` による音声同期・再生は継続する（＝レイヤーを隠しても音は鳴り続ける）。
5. クリップ種別ごとに描画関数へ分岐する（下表）。
6. 各描画は `compCtx.save()/restore()` で状態を分離しつつ、`globalAlpha`（不透明度×フェード係数×エフェクト由来の透明度）・`globalCompositeOperation`（ブレンドモード）・`filter`（明るさ/コントラスト/彩度＋ぼかし）を設定 → クリップ中心基準で `translate → rotate → scale` → `drawImage`/`fillText` という順で描く。

| 種別 | 描画関数 | 内容 |
| ------ | ---------- | ------ |
| 動画/画像 | `drawVisualClip` | `drawImage` で描画 |
| テキスト | `drawTextClip` | `fillText`/`strokeText`（YMM4書式再現） |
| 図形 | `drawShapeClip` | 現状は背景単色塗りのみ対応 |
| エフェクトアイテム | `drawEffectOverlay` | それまでに描画済みの下位レイヤーへ全画面効果（放射光など）を`lighter`合成で加算 |
| 音声 | （描画なし） | `syncAV` のみ |

#### ブレンドモード

YMM4のブレンドモード名を `BLEND_MODES` テーブルで Canvas の `globalCompositeOperation` に変換して再現している（対応済み: 通常/加算/乗算/スクリーン/オーバーレイ/比較(暗)/比較(明)/差分/除外/覆い焼き/焼き込み/ハードライト/ソフトライト）。

#### アニメーション・エフェクトの評価

- YMM4のキーフレームアニメーション（位置・スケール・回転・不透明度）は事前ベイクせず、**毎フレーム `evalAnimValues()` が進行度 p（0〜1）から値を都度補間**して描画時に算出する。対応する補間種別は「直線移動」「瞬間移動（区間で値が切り替わる）」「加減速移動（smoothstep 近似）」の3種。
- 登場/退場エフェクト（`move-outside` / `inout-blur` / `crash`）やランダム移動エフェクトも同様に `effectState()` が毎フレーム位置・ぼかし量・拡大率・透明度のオフセットを算出する（ランダム移動は疑似乱数 `prand()` をクリップID由来のシードで固定し、同じフレームなら常に同じ結果になるようにしている）。
- クロマキーは「縮小オフスクリーンcanvasへ描画 → `getImageData` でキー色とのRGB距離を判定 → 該当ピクセルのアルファを0にして `putImageData` → 合成用ソースとして描画元を差し替え」というピクセル単位のCPU処理。GPUシェーダーを使わない代わりに処理幅を最大640pxへ縮小して負荷を抑えている。

#### 音声再生エンジン

- 動画/音声クリップは参照された時点で遅延生成される `<video>`/`<audio>` 要素（`getMediaEl`）を1つずつ持つ。**これらの要素はDOMツリーには追加されず**、`drawImage` によるフレーム取得（動画の場合）とブラウザの内蔵デコーダ・オーディオ出力のためだけに保持される、いわば「見えないメディア要素」。
- `syncAV(clip, time, playing, trackId)` が毎フレーム呼ばれ、クリップの`offset`（トリム・分割で進んだ素材内位置）と再生ヘッドから理論上の素材内時刻を計算し、実際の `currentTime` とのズレが **0.35秒を超えたときだけ** `currentTime` を補正する（毎フレームseekするとガタつくため許容誤差を設けている）。
- 音量は「クリップ音量 × レイヤー音量（`LayerSettings.Volume`）× フェード係数 × マスター音量」の掛け合わせ。YMM4はクリップ音量が100%を超えられる（最大172.5%など）ため、`AudioContext` の `GainNode` を `MediaElementSource` に接続してWebAudio経由で増幅する（`el.volume` は0〜1の範囲しか取れないため）。WebAudio接続に失敗した場合（未対応環境など）は `el.volume` のみでのフォールバック制御になる。
- ブラウザの自動再生制限対策として、`AudioContext` は再生ボタン押下（ユーザー操作起点）で `resume()` される。
- そのフレームで非アクティブになったクリップは `pauseInactive()`、全体停止時は `pauseAllMedia()` で一時停止され、裏で鳴り続けないようにしている。

#### 書き出し（ファイルレンダリング）について

- 現状は上記のプレビュー合成のみで、**動画ファイルへの書き出しエンジンは未実装**（`MediaRecorder` / ffmpeg 等は不使用）。可能なのは右クリック「現在フレームを保存」による `canvas.toDataURL('image/png')` の単一フレームPNG書き出しのみ。
- 将来的な書き出し機能は §6-3 のとおり `canvas + MediaRecorder` 等を用いた実装を想定（未着手）。

## 3. 実装済み機能 ✅

### 3-1. タイムライン編集

- **連番レイヤー方式**: YMM4 ライクに種類を設けず「レイヤー1〜5」。どのレイヤーにも映像・音声・テキストを配置可（YMM4 と同じく下の行ほど手前に描画）。
- クリップの**ドラッグ移動 / 左右ハンドルでのリサイズ（トリミング）**。
- **分割（カットツール / 分割ボタン / `S`）** — 再生ヘッド位置でクリップを2分割。
- **削除**（ボタン / `Delete` / `Backspace`）。
- **複製（`Ctrl+D`）/ コピー（`Ctrl+C`）/ 貼り付け（`Ctrl+V`）**。
- ツール切替: 選択(`V`) / カット(`C`) / ハンド(`H`)。
- **ズーム**スライダー（ルーラー・クリップ幅が連動）。
- **ルーラー / 再生ヘッド（赤線）** — どこでも掴んでドラッグスクラブ可能。空白クリックでもシーク。
- 横スクロールで再生ヘッド連動、縦スクロールでレイヤーヘッダー連動。
- タイムライン領域の**高さをドラッグでリサイズ**（`#tlResizer`・localStorage 保存）。

### 3-2. メディア

- **読み込み**: ボタン / ファイル入力 / **OS からのドラッグ&ドロップ**（直接トラックへ配置可）。
- 対応形式: 動画(mp4/mov/webm/mkv)、音声(mp3/wav/m4a)、画像(png/jpg/gif/webp)。
- **サムネイル自動生成**（動画は中間フレームを canvas 描画）。
- **再生尺の自動取得**（`probeDuration`）でバッジ表示・クリップ長に反映。
- メディアの右クリックメニュー: モニター表示 / タイムライン追加 / 名前変更 / プールから削除。

### 3-3. プレビュー・再生

- **canvas コンポジター**による複数レイヤー合成プレビュー。
- 再生 / 一時停止（`Space`）、先頭・末尾、前後フレーム、**ループ**、音量。
- 動画・音声のタイムライン位置同期再生（`syncAV` / 再生速度・ミュート対応）。
- **解像度切替**（1080p / 縦型 / 正方形 / 4K）— canvas とアスペクト比に反映。
- **タイムコード表示**（HH:MM:SS:FF / 30fps）。
- 右クリック「現在フレームを保存」で PNG ダウンロード。

### 3-4. プロパティパネル

選択クリップに対しリアルタイム反映（合成へ即時反映）:

- 変形（位置 X/Y・スケール・回転）
